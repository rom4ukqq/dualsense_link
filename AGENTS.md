# AGENTS.md

## Правила разработки

1. Соблюдать принципы `SOLID`, `DRY`, `KISS`.
2. Делать аккуратно: писать чистый, читаемый и поддерживаемый код.
3. Размер файла: не более `350` строк (целевой диапазон `300-350` максимум).
4. Размер метода/функции: не более `40` строк (целевой диапазон `30-40` максимум).

## Практика применения

1. Если файл разрастается, выносить логику в отдельные модули.
2. Если метод растет, разбивать на небольшие приватные/вспомогательные методы.
3. Избегать дублирования, повторно использовать существующие абстракции.
4. Предпочитать простые решения без лишней сложности.
